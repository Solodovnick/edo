## ✅ РЕШЕНИЕ: ЕДИНАЯ СИСТЕМА ДАННЫХ

### Цель:
Создать **ЕДИНЫЙ пул обращений**, который будет использоваться во всех трёх кабинетах с разными представлениями.

### Общая структура:
```typescript
interface UnifiedAppeal {
  // БАЗОВЫЕ ПОЛЯ (для всех)
  id: string
  regDate: string
  category: 'Письменное' | 'Устное'
  status: string
  deadline: string
  responsible: string
  applicantName: string
  organizationName: string
  address: string
  type: 'Физ лицо' | 'Юр лицо' | 'Регулятор'
  
  // КОНТЕНТ (для всех)
  content: string          // Исходное обращение
  solution: string         // Текст решения
  response: string         // Форма ответа
  
  // СПЕЦИФИЧНЫЕ ПОЛЯ
  phone?: string           // Телефон (для аудитора и детальных карточек)
  cbs?: string            // Банк (для ответственного)
  isMine?: boolean        // Фильтр (для ответственного)
  
  // АУДИТ
  auditStatus?: 'pending' | 'approved' | 'rejected'
  deadlineCountdown?: { days: number, hours: number, minutes: number }
  
  // ДОПОЛНИТЕЛЬНЫЕ ДАННЫЕ
  attachments?: Array<{ id: number, name: string, date: string }>
  history?: Array<{ number: string, date: string, status: string }>
  crmComments?: Array<{ author: string, date: string, text: string }>
  
  // МЕТАДАННЫЕ
  createdAt?: string
  updatedAt?: string
}
```
