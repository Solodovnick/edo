# Panel Toggle Feature - Управление панелями Регистратора

## ✅ Реализовано: 01.03.2026

### Описание функции:

Добавлены кнопки управления видимостью панелей на странице Регистратора:
- **"Скрыть/Показать форму"** - управляет левой панелью с формой регистрации
- **"Скрыть/Показать превью"** - управляет правой панелью с превью обращения

Это даёт пользователю гибкость в организации рабочего пространства.

## 💾 State Management

### Добавлены состояния:
```typescript
const [showForm, setShowForm] = useState(true);      // Форма видима по умолчанию
const [showPreview, setShowPreview] = useState(true); // Превью видимо по умолчанию
```

### Логика переключения:
```typescript
// Кнопка "Форма"
<button onClick={() => setShowForm(!showForm)}>
  {showForm ? <PanelLeftClose /> : <PanelLeftOpen />}
  {showForm ? 'Скрыть форму' : 'Форма'}
</button>

// Кнопка "Превью"
<button onClick={() => setShowPreview(!showPreview)}>
  {showPreview ? <PanelRightClose /> : <PanelRightOpen />}
  {showPreview ? 'Скрыть превью' : 'Превью'}
</button>
```

## 📐 Responsive Layout

### Адаптивная сетка:
```typescript
className={`flex-1 overflow-hidden grid gap-0 ${
  !showForm && !showPreview ? 'grid-cols-1' :  // Обе скрыты
  !showForm ? 'grid-cols-1' :                  // Только превью
  !showPreview ? 'grid-cols-1' :               // Только форма
  'grid-cols-1 lg:grid-cols-2'                 // Обе видимы
}`}
```

### Условный рендеринг:
```typescript
{/* LEFT: Form */}
{showForm && (
  <div className="flex-1 overflow-y-auto p-2 md:p-3">
    {/* Форма... */}
  </div>
)}

{/* RIGHT: Preview */}
{showPreview && (
  <div className={`flex flex-col p-6 lg:p-8... ${
    showForm ? 'hidden lg:flex' : ''  // Скрыть на мобильных если форма открыта
  }`}>
    {/* Превью... */}
  </div>
)}
```

## 🎯 Use Cases

### UC-1: Фокус на форме
```
Сценарий: Пользователь заполняет длинную форму
1. Нажимает "Скрыть превью"
2. Форма расширяется на весь экран
3. Больше места для полей ввода
4. Меньше отвлекающих факторов
5. После заполнения - "Показать превью" для проверки
```

### UC-2: Проверка данных
```
Сценарий: Пользователь хочет проверить заполненное обращение
1. Форма уже заполнена
2. Нажимает "Скрыть форму"
3. Превью расширяется на весь экран
4. Большая карточка с детальной информацией
5. Легче проверить все данные
```

### UC-3: Демонстрация клиенту
```
Сценарий: Оператор показывает клиенту зарегистрированное обращение
1. Нажимает "Скрыть форму"
2. Превью на весь экран
3. Озвучивает номер обращения
4. Показывает клиенту все детали
5. Клиент видит категорию, приоритет, SLA
```

### UC-4: Мобильная работа
```
Сценарий: Планшет или маленький ноутбук
1. По умолчанию split screen может быть тесным
2. Пользователь переключается между панелями
3. Заполняет форму → Скрывает превью
4. Проверяет данные → Скрывает форму
5. Гибкий workflow без скролла
```

## 🔧 Технические детали

### Импортированные иконки:
```typescript
import {
  PanelLeftClose,   // Скрыть левую панель (форму)
  PanelLeftOpen,    // Показать левую панель
  PanelRightClose,  // Скрыть правую панель (превью)
  PanelRightOpen    // Показать правую панель
} from 'lucide-react';
```

### Claymorphism стиль кнопок:
```typescript
style={{
  boxShadow: `
    inset -2px -2px 6px rgba(255, 255, 255, 0.7),  // Светлая часть (объём)
    inset 2px 2px 6px rgba(0, 0, 0, 0.15),         // Тёмная часть (глубина)
    0 6px 15px rgba(201, 231, 253, 0.4)            // Внешняя тень (парение)
  `
}}
```

### Hover эффект:
```typescript
className="... hover:scale-105 transition-transform"
```

При наведении кнопка немного увеличивается для интерактивности.

## 📱 Responsive Design

### Desktop (≥ 1024px):
```
- Обе панели: Split screen 50/50
- Только форма: Форма на 100% ширины
- Только превью: Превью на 100% ширины
- Текст кнопок: Полный ("Скрыть форму" / "Скрыть превью")
```

### Tablet (768px - 1024px):
```
- Обе панели: Стакинг (форма сверху, превью снизу)
- Только форма: Форма на 100%
- Только превью: Превью на 100%
- Текст кнопок: Сокращённый ("Форма" / "Превью")
```

### Mobile (< 768px):
```
- Обе панели: Только форма видна (превью hidden)
- Можно переключаться кнопками
- Текст кнопок: Только иконки (без текста)
- Компактные кнопки: px-3 py-2
```

## 📊 Метрики использования (прогноз)

### Ожидаемые паттерны:
```
70% - Обе панели видимы (по умолчанию)
20% - Только форма (фокус на заполнении)
8%  - Только превью (проверка/демонстрация)
2%  - Переключение между режимами
```

### Типичный workflow:
```
1. Открыть страницу → Обе панели
2. Начать заполнение → Скрыть превью (фокус)
3. Заполнить форму → Показать превью (проверка)
4. Зарегистрировать → Скрыть форму (демонстрация клиенту)
5. Новое обращение → Показать форму + Сбросить
```
