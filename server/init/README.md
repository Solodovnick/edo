# Инициализация PostgreSQL (Docker)

Файлы в этой папке монтируются в `/docker-entrypoint-initdb.d` и выполняются **один раз** при создании пустого volume.

## `02-app-dictionary-and-appeals.sql`

- Схема **`app`**: справочники `dict_appeal_status`, `dict_applicant_type`, `dict_appeal_channel` (связь статусов с кабинетами через массив `cabinets`) и единая таблица **`app.appeal_card`** (`id`, `status_code` → словарь, `data` JSONB — тело как у REST `AppealDto`).
- Используется Node-сервером (`server/edoPgStore.mjs`) при заданном **`DATABASE_URL`** в `.env`.
- Если volume Postgres уже создан **до** появления этого файла, накатите вручную: `npm run db:init-app` (контейнер `edo-postgres` должен быть запущен).

## `01-edo-schema.sql`

- Полная схема из ERD (таблицы `appeals`, `attachments`, `audit`, …).
- В экспорте pgAdmin не было `CREATE TYPE` для enum — добавлены типы: `applicant_type_enum`, `appeal_type_enum`, `appeal_priority_enum`, `incident_status_enum` (значения см. в начале файла).
- Порядок `CREATE TABLE` выстроен по зависимостям внешних ключей (в отличие от произвольного порядка в ERD).
- Для полей `id` первичных ключей добавлено **`GENERATED BY DEFAULT AS IDENTITY`**, чтобы можно было вставлять строки без явного `id` (в сыром DDL были только `bigint NOT NULL`).
- Добавлен внешний ключ **`users.role_id` → `roles(id)`** (в ERD не был явно указан).

Чтобы пересоздать БД с нуля:

```bash
cd edo && docker compose down -v && docker compose up -d
```

## Восстановление из бэкапа (`*.sql`)

Учётные данные контейнера **edo** (см. `docker-compose.yml`): пользователь **`edo`**, пароль **`edo`**, БД **`edo`**, порт **`5432`**.

### 1. Запусти Postgres

```bash
cd edo
docker compose up -d
```

Подожди несколько секунд, пока контейнер станет healthy.

### 2. Накати бэкап в базу `edo`

**Вариант A — через `docker exec` (не нужен локальный `psql`):**

```bash
docker exec -i edo-postgres psql -U edo -d edo < /Users/solodaleks/Downloads/backup.sql
```

Путь к файлу замени на свой, если бэкап лежит в другом месте.

**Вариант B — если установлен `psql` на машине:**

```bash
export PGPASSWORD=edo
psql -h 127.0.0.1 -p 5432 -U edo -d edo -f /Users/solodaleks/Downloads/backup.sql
```

### 3. Проверка

```bash
docker exec -it edo-postgres psql -U edo -d edo -c "\dt"
```

или открой приложение с `npm run dev` и посмотри индикатор PostgreSQL в шапке.

### Важно

- Если в `backup.sql` есть **`CREATE DATABASE`** / подключение к другой БД — либо правь скрипт под **`edo`**, либо восстанавливай в ту БД, которую задаёт бэкап (тогда создай пользователя/БД вручную или поменяй `POSTGRES_DB` в compose).
- Если бэкап **конфликтует** с уже накатанным `server/init/*.sql` (дубли таблиц), проще **чистый volume** и потом только бэкап:
  ```bash
  cd edo && docker compose down -v && docker compose up -d
  # подождать готовности Postgres, затем:
  docker exec -i edo-postgres psql -U edo -d edo < /Users/solodaleks/Downloads/backup.sql
  ```
  Либо временно убери монтирование `./server/init` из `docker-compose.yml`, если init должен полностью замениться дампом.
- Дамп в формате **custom** (`pg_restore`, файл `.dump`) — это не `psql -f`; нужен `pg_restore` (отдельно могу описать, если формат другой).
