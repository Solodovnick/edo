## FR-01 Регистрация обращений

*Приоритет: P0*

| ID | Требование | Приоритет |
|---|---|---|
| FR-01.1 | Система поддерживает три категории заявителей: физическое лицо, юридическое лицо, регулятор (ЦБ РФ) | P0 |
| FR-01.2 | Система поддерживает **типы** обращения: письменное, устное, регуляторное (отдельный контур SLA и полей) — см. `docs/business-requirements.md`, словарь данных | P0 |

| FR-01.5 | Форма регистрации валидируется перед сохранением: обязательные поля — категория заявителя, тип, текст обращения | P0 |

## FR-02 Жизненный цикл обращения

*Приоритет: P0*

| ID | Требование | Приоритет |
|---|---|---|
| FR-02.1 | Обращение проходит статусы по модели жизненного цикла в **`docs/state-diagram.md`** (черновик → регистрация → работа → решение → аудит при необходимости → закрытие → архив). Конкретные имена статусов не должны противоречить справочнику заказчика | P0 |
| FR-02.2 | Переход статуса выполняется только уполномоченным пользователем (регистратор, ответственный, аудитор, руководитель, администратор — по матрице UC и назначению роли) | P0 |
| FR-02.3 | Система фиксирует историю переходов статусов с временными метками и автором | P0 |
| FR-02.4 | Ответственный может переназначить обращение другому специалисту | P1 |
| FR-02.5 | При эскалации обращения уведомляется вышестоящий руководитель | P1 |
