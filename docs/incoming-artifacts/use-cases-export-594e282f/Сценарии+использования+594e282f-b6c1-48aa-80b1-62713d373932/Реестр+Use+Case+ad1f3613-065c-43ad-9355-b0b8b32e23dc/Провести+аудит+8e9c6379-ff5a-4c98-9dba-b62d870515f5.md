|**ID:** Au.02|
|-|
|**Название:** Провести аудит|
|**Действующее лицо:** Ответственный Аудитор, Система|
|**Предусловия:** 
Если хотя бы одно обращение в статусе "Решено"
Обращение было распределено автоматически из пула ответственному аудитору со статусом "Аудит" (Au-xxxx)
**Триггер:** Пользователь нажимает на кнопку "Выполнить аудит"
**Постусловие:** Открыта карточка Обращения|
|**Основной поток: 
1.** Система отображает поле "Форма обратной связи"
2. Пользователь нажимает на выпадающий список  "Выберите шаблон"
3. Система отображает варианты шаблонов в выпадающем списке
4. Пользователь нажимает на один из вариантов шаблона
5. Система отображает в поле "Форма обратной связи" текстовые поля с вопросами из шаблона и флажки "Оценка от 1 до 5 *" 
6. Пользователь заполняет атрибуты для текстовых полей с вопросами
7. Пользователь выбирает оценку заявителя и нажимает на флажок с цифрой
8 Пользователь заполняет атрибуты поля дозвон
9. Пользователь нажимает кнопку "Дозвонился"
10. Система отображает поле "Аудит успешно завершен! Статус обращения изменен на "В архиве".
11. Пользователь нажимает кнопку "ОК"
12. Система меняет статус обращения на "В архиве"
13. Система перемещает обращение в архив
14. Сценарий окончен|
|**Расширение**:
10а1. Опубликован и подтвержден результат аудита с оценкой 1-3
10а2. Система меняет статус обращения на "В архиве"
10а3. Система отправляет уведомление Ответственному по обработке и Руководителю
10а4. Система меняет статус обращения на "Пройден аудит"
10а5. Система перемещает обращение в архив
10а6. Сценарий окончен

10б1. В карточке аудит обращение есть незаполненные обязательные поля и атрибуты
10б2. Пользователь нажимает Опубликовать результаты аудита
10б3. Система выдает ошибку "не все обязательные поля заполнены"
10б4. Пользователь подтверждает, карточка обращения доступна для заполнения
10б5. Сценарий окончен|
