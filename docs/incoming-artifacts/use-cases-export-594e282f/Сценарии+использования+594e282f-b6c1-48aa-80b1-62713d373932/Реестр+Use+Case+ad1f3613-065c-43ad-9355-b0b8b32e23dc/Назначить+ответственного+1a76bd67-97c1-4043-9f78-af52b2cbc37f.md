**Предусловия:**

Пользователь имеет права на переадресацию обращения.

Ответственный инициировал отказ от обращения.

Статистика нагрузки исполнителей актуальна (загружена <5 мин назад).




**Триггер:** Пользователь получил Alert с Уведомлением на отказ от обращения



**Основной поток**

1. Пользователь дает команду Просмотреть Alert

2. Система отображает форму Карточка обращения следующими полями:[содержание] обращения; [категория обращения];[комментарий ответственного]; [причина отказа].

3. Система дает возможность Принять отказ, Отклонить отказ.

4. Пользователь дает команду Принять отказ.

5. Система запрашивает подтверждение команды Принять отказ.

6. Пользователь подтверждает команду Принять отказ.

7. Система редактирует [Сотрудника] с Сотрудник.ID == текущий пользователь со следующими атрибутами Сотрудник.Активность="не участвует в распределении".

8. Система дает возможность Назначить распределение автоматически или Назначить вручную

9. Пользователь дает команду Назначить вручную

10. Система формирует список сотрудников с учетом нагрузки, категории, и рабочего графика (ФТ код_ФТ).

11. Система отображает список найденных сотрудников (ФТ код ФТ).

12. Система дает возможность выбрать ответственного

13. Пользователь дает команду Выбрать ответственного

14. Система редактирует [Сотрудника] с Сотрудник.Назначен_На со следующими атрибутами

- Обращение *ID = ID*_текущего обращения

- *Сотрудник.ID = ID*_выбранного сотрудника

1. Система уведомляет Ответственного о назначении нового обращения.

2. Система уведомляет Ответственного о снятии обращения  с обработки

3. Сценарий окончен.



**Расширения:** 

4а. Пользователь дает команду Отклонить отказ.

4а.1 Система запрашивает подтверждение команды Отклонить отказ.

4а.2 Пользователь подтверждает команду Отклонить отказ.

4а.3 Сценарий окончен.



6а Пользователь не подтверждает  команду Принять отказ.

6а.1 Сценарий возвращается на шаг 3 Основного потока.



9а Пользователь дает команду Назначить распределение автоматически

9а.1Система предлагает назначить задачу на сотрудника [ФИО] с учетом текущей нагрузки, категории,   и рабочего графика (ФТ код_ФТ)..

9а.2 Система запрашивает подтверждение команды Назначить задачу на сотрудника [ФИО].

9а.3 Пользователь подтверждает  команду.

9а.4  Сценарий переходит на шаг 14 Основного потока.



9а.3а Пользователь не подтверждает команду.

9а.3а.1 Система уведомляет руководителя: "Выбор ответственного не выполнен. Обращение ожидает назначения в Кабинете руководителя".

9а.3а.2 Сценарий переходит на шаг 8 Основного потока.



12а Нет свободных сотрудников для ручного распределения.

12а.1 Сценарий возвращается на шаг 8 Основного потока.



13а Руководитель не выбрал ответственного в течение 5 минут

13а.1 Система уведомляет руководителя: "Выбор ответственного не выполнен. Обращение ожидает назначения в Кабинете руководителя".

13а.1 Сценарий переходит на шаг 8 Основного потока.



**Постусловие:** 



Обращение имеет нового Ответственного (Сотрудник. ID обновлен) 

Предыдущий Ответственный снят с обработки (Активность="не участвует") 

В audit trail: "Смена ответственного [ID] → [ID] [дата]" 

Руководитель уведомлен о результате (если требовалось)
