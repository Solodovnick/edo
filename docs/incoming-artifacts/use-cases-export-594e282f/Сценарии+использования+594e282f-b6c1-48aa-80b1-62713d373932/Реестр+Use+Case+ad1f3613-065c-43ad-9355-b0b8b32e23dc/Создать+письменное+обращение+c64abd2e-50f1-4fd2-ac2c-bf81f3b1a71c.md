|**Автор**: Расул|
|**Название: Создать письменное обращение**|
|**Действующее лицо: Регистратор**|
|**Предусловия:** регистратор авторизовался в системе
**Триггер:** в модуле внешней корреспонденции зарегистрировано письмо с обращением заявителя
**Постусловие:** 
зарегистрировано письменное обращение, 
обращение отправлено в пул для распределения на ответственного, 
в CRM появилась информация об обращении (если оно от клиента банка)|
|**Основной поток:**
1. Система получает из модуля внешней корреспонденции данные для создания обращения;
2. Система создаёт обращение: номер, дата и время, статус, вложение (отсканированное письмо);
3. Система назначает регистратора;
4. Регистратор получает уведомление о назначенном письменном обращении;
5. Регистратор даёт команду открыть обращение;
6. Система открывает обращение;
7. Регистратор просматривает данные по обращению;
8. Регистратор выбирает тип обращения и тип заявителя;
9. Система отображает поля обращения в соответствии с выбранными типами;
10. Регистратор заполняет обязательные для регистрации обращения данные;
11. Система обращается к CRM для проверки данных заявителя;
12. Система проверяет является ли заявитель клиентом банка и есть ли у него персональный менеджер;
13. Система НЕ нашла данные о заявителе в CRM;
14. Система проверяет историю обращений заявителя;
15. Система НЕ нашла историю обращений в обращение;
16. Система отображает уведомление о выполнении проверок;
17. Регистратор заполняет категорию и подкатегорию обращения;
18. Система заполняет приоритет обращения;
19. Регистратор заполняет прочие данные об обращении;
20. Регистратор регистрирует обращение;
21. Система проверяет заполненность всех обязательных для регистрации обращения полей;
22. Система сохраняет внесённую информацию;
23. Система уведомляет пользователя об успешной регистрации обращения;
24. Система обращается к CRM для передачи информации об обращении;
25**. Сценарий окончен**|
|**Расширение**:
13а. Заявитель клиент без персонального менеджера
13а.1 Система заполняет признак клиент, но не данные о персональном менеджере;
13а.2 Сценарий переходит к шагу 14

13б. Заявитель клиент с персональным менеджером
13б.1 Система заполняет признак клиент и данные о персональном менеджере;
13б.2 Сценарий переходит к шагу 14

15а. У заявителя есть история обращений
15а.1 Система добавляет в обращение историю обращений заявителя;
15а.2 Сценарий переходит к шагу 16

17а. Проверка регистратором базы знаний по категориям и подкатегориям обращений
17а.1 Регистратор даёт команду открыть базу знаний по категориям и подкатегориям;
17а.2 Система открывает базу знаний;
17а.3 Регистратор просматривает список категорий и подкатегорий;
17а.4 Регистратор даёт команду открыть информацию по категории/подкатегории;
17а.5 Система отображает информацию по выбранной категории/подкатегории;
17а.6 Регистратор даёт команду закрыть базу знаний по категориям и подкатегориям;
17а.7 Система закрывает базу знаний по категориям и подкатегориям и возвращает регистратора обратно к обращению;
17а.8 Регистратор заполняет категорию и подкатегорию обращения;
17а.9 Сценарий переходит к шагу 18

21а. Не все обязательные поля заполнены
21а.1 Система уведомляет регистратора о незаполненных обязательных полях;
21а.2 Регистратор дозаполняет обращение;
21а.3 Регистратор регистрирует обращение;
21а.4 Сценарий переходит к шагу 22

24а. Данные об обращении не удалось передать в CRM
24а.1 Система не получила подтверждение, что данные приняты;
24а.2 Система обращается в CRM каждый час пока не получит ответ, есть ли в CRM обращение с определённым id:
24а.2.1 Если есть, то обращения прекращаются;
24а.2.2 Если нет, то отправляет данные об обращении повторно.
**24а.3 Сценарий окончен.**|
