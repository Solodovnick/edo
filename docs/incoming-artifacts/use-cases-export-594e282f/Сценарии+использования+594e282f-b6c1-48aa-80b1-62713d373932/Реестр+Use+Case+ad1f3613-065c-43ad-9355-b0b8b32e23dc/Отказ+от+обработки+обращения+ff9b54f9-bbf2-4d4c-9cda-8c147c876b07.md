|**ID:** Au.02|
|-|
|**Название:** Отказ от обработки обращения|
|**Действующее лицо:** Ответственный Аудитор, Руководитель, Система|
|**Предусловия:** 
Если хотя бы одно обращение в статусе "Решено" (UC-XXX)
Обращение было распределено автоматически из пула ответственному аудитору (Au-xxxx)
**Триггер:** Пользователь дал команду "Взять в работу"
**Постусловие:** Обращение передано в пул обращений|
|**Основной поток: 
1.** Пользователь (Ответственный аудитор) в карточке обращения нажимает кнопку «Отказаться».
2. Система отображает форму с полями: «Объяснительная причина» (текстовое поле) и кнопками «ОК» и «Отмена».
3. Пользователь выбирает шаблон объяснительной (если предусмотрено) и заполняет поле «Причина».
4. Пользователь нажимает кнопку ОК
5. Система создаёт уведомление руководителю с текстом объяснительной причины.
6. Система **отправляет уведомление** Ответственному аудитору
7. Сценарий окончен|
|**Расширение**:
2a. Пользователь нажимает «Отмена» — система закрывает форму, отказ не выполняется.|
