**Действующее лицо:** Ответственный исполнитель 
**Предусловия:** Исполнитель успешно аутентифицирован в системе (присутствует валидный JWT токен) и имеет соответствующую роль.
**Триггер:** Исполнитель дает команду открыть раздел "Кабинет ответственного" в системе.
**Постусловия:** На экране отображен список обращений, доступных исполнителю и отсортированных/отфильтрованных по заданным критериям. Данные поиска сохранены в текущей сессии.

**Основной поток:**

1. Исполнитель дает команду открыть Кабинет ответственного;

2. Система проверяет авторизацию (JWT) и роль исполнителя для применения политик видимости (RLS);

3. Система загружает очередь "Мои обращения" с сортировкой по "Дедлайну (ASC)" (ближайшие SLA первыми) и лимитом в 50 записей;

4. Система подписывается на обновления реестра (через WebSocket) для актуализации статусов в реальном времени;

5. Система отображает реестр обращений, счетчики по статусам и цветовые индикаторы SLA;

6. Исполнитель вводит произвольный текст для быстрого поиска или выбирает дополнительные фильтры (тип, заявитель, статус, ответственность);

7. Система валидирует введенные критерии поиска (проверяет формат номера OBR, длину ИИН/БИН и дат);

8. Система выполняет поиск обращений в БД, комбинируя фильтры по логике AND;

9. Система логирует поисковый запрос в аудит-лог;

10. Система сохраняет параметры поиска, фильтров и пагинации в локальную сессию (sessionStorage);

11. Система обновляет список найденных обращений и счетчики результативности;

12. Исполнитель дает команду открыть карточку конкретного обращения;

13. Система переходит в режим просмотра выбранного обращения;

14. Сценарий окончен.

**Расширения:**

3а. У исполнителя нет назначенных в работу обращений

3а.1 Система отображает пустой реестр с заглушкой "У вас пока нет назначенных обращений";

3а.2 Действия по поиску остаются доступными. Сценарий переходит к шагу 6.

6а. Исполнитель запрашивает фильтр "Все обращения"

6а.1 Система выводит предупреждение об ограниченном доступе;

6а.2 Система применяет фильтр только для обращений, относящихся к подразделению исполнителя;

6а.3 Сценарий переходит к шагу 8.

7а. Введенные критерии (например, ИИН или даты) не прошли валидацию

7а.1 Система уведомляет исполнителя об ошибке и подсвечивает некорректно заполненные поля;

7а.2 Исполнитель корректирует критерии поиска;

7а.3 Сценарий возвращается к шагу 7.

8а. БД недоступна или запрос превышает таймаут

8а.1 Система уведомляет о технической ошибке или превышении времени ожидания, предлагая обновить страницу или сузить поиск;

8а.2 Исполнитель дает команду обновить страницу и сужает поиск;

8а.3 Сценарий возвращается к шагу 7.

11а. По заданному набору критериев обращения не найдены

11а.1 Система отображает уведомление "🔍 По вашему запросу ничего не найдено" и предлагает рекомендации по изменению фильтров;

11а.2 Исполнитель очищает или корректирует критерии поиска;

11а.3 Сценарий возвращается к шагу 7.

11б. Результатов поиска слишком много (более 500)
11б.1 Система уведомляет исполнителя о слишком большом количестве найденных записей;
11б.2 Система отображает только первые 500 записей и предлагает уточнить параметры поиска;
11б.3 Сценарий переходит к шагу 12 (или исполнитель уточняет критерии - шаг 6).
