|**Автор**: Расул|
|**Название: Создать устное обращение**|
|**Действующее лицо: Регистратор**|
|**Предусловия:** регистратор авторизовался в системе
**Триггер:** в банк по телефону или лично обратился заявитель
**Постусловие:** 
зарегистрировано устное обращение, 
обращение отправлено в пул для распределения на ответственного, 
в CRM появилась информация об обращении (если оно от клиента банка)|
|**Основной поток:**
1. Регистратор даёт команду открыть форму нового обращения;
2. Система открывает новое обращение;
3. Регистратор выбирает тип обращения и тип заявителя;
4. Система отображает поля обращения в соответствии с выбранными типами;
5. Регистратор заполняет обязательные для регистрации обращения данные;
6. Система обращается к CRM для проверки данных заявителя;
7. Система проверяет является ли заявитель клиентом банка и есть ли у него персональный менеджер;
8. Система НЕ нашла данные о заявителе в CRM;
9. Система проверяет историю обращений заявителя;
10. Система НЕ нашла историю обращений в обращение;
11. Система отображает уведомление о выполнении проверок;
12. Регистратор заполняет категорию и подкатегорию обращения;
13. Система заполняет приоритет обращения на основании типа обращения;
14. Регистратор заполняет прочие данные об обращении;
15. Регистратор регистрирует обращение;
16. Система проверяет заполненность всех обязательных для регистрации обращения полей;
17. Система сохраняет внесённую информацию;
18. Система уведомляет пользователя об успешной регистрации обращения;
19. Система обращается к CRM для передачи информации об обращении;
20**. Сценарий окончен.**|
|**Расширение**:
8а. Заявитель клиент без персонального менеджера
8а.1 Система заполняет признак клиент, но не данные о персональном менеджере;
8а.2 Сценарий переходит к шагу 9

8б. Заявитель клиент с персональным менеджером
8б.1 Система заполняет признак клиент и данные о персональном менеджере;
8б.2 Сценарий переходит к шагу 9

10а. У заявителя есть история обращений
10а.1 Система добавляет в обращение историю обращений заявителя;
10а.2 Сценарий переходит к шагу 11

12а. Проверка регистратором базы знаний по категориям и подкатегориям обращений
12а.1 Регистратор даёт команду открыть базу знаний по категориям и подкатегориям;
12а.2 Система открывает базу знаний;
12а.3 Регистратор просматривает список категорий и подкатегорий;
12а.4 Регистратор даёт команду открыть информацию по категории/подкатегории;
12а.5 Система отображает информацию по выбранной категории/подкатегории;
12а.6 Регистратор даёт команду закрыть базу знаний по категориям и подкатегориям;
12а.7 Система закрывает базу знаний по категориям и подкатегориям и возвращает регистратора обратно к обращению;
12а.8 Регистратор заполняет категорию и подкатегорию обращения;
12а.9 Сценарий переходит к шагу 13

16а. Не все обязательные поля заполнены
16а.1 Система уведомляет регистратора о незаполненных обязательных полях;
16а.2 Регистратор дозаполняет обращение;
16а.3 Регистратор регистрирует обращение;
16а.4 Сценарий переходит к шагу 17

19а. Данные об обращении не удалось передать в CRM
19а.1 Система не получила подтверждение, что данные приняты;
19а.2 Система обращается в CRM каждый час пока не получит ответ, есть ли в CRM обращение с определённым id:
19а.2.1 Если есть, то обращения прекращаются;
19а.2.2 Если нет, то отправляет данные об обращении повторно.
**19а.3 Сценарий окончен.**|
