|**Название:** Решить обращение досрочно|
|**Действующее лицо: Регистратор**|
|**Предусловия:** создано устное обращение
**Триггер:** регистратор вызвал функцию досрочного решения обращения
**Постусловие:** устное обращение решено, обращение отправлено в пул для распределения на аудитора, в CRM появилась информация об обращении (если оно от клиента банка)|
|**Основной поток:**
1. Регистратор даёт команду решить обращение досрочно;
2. Система отображает поля для досрочного решения обращения ([Обращение]: Решение, Шаблон ответа, Ответ);
3. Регистратор регистрирует обращение;
4. Система проверяет заполненность всех обязательных для регистрации и решения обращения полей;
5. Система сохраняет внесённую информацию;
6. Система уведомляет пользователя об успешной регистрации обращения;
7. Система обращается к CRM для передачи информации об обращении;
8**. Сценарий окончен** |
|**Расширение**:
4а. Не все обязательные поля заполнены
4а.1 Система уведомляет регистратора о незаполненных обязательных полях;
4а.2 Регистратор дозаполняет обращение;
4а.3 Регистратор регистрирует обращение;
4а.4 Сценарий переходит к шагу 5|
