# Статусы обращения — из выгрузки SAB Group WIN'26 (CSV)

**Источник:** первая таблица в [`SAB-Group-WIN26-EDO-context-export.csv`](SAB-Group-WIN26-EDO-context-export.csv).  
**Назначение:** быстрая сверка с [`docs/state-diagram.md`](../../state-diagram.md); расхождения по формулировкам и орфографии в исходнике сохранены там, где это не мешает идентификации кода.

## Основной контур статусов

| Код статуса | Поведение (как в импорте) |
|-------------|---------------------------|
| DRAFT | Номер обращению не присвоен, карточка уже заполняется |
| REGISTERED | Номер обращению присвоен, отвественный ещё не назначен |
| IN PROGRESS | Отвественный назначен |
| UNDER REVIEW | Ответ на письменное обращение отправлен на подпись руководителю |
| SIGNED | Ответ подписан рукооодителем |
| CLOSED | Обработка обращения завершена |
| AUDIT | Назначен аудитор на устное обращение |
| IN ARCHIVE | Обращение попало в архив |

## Доп. статус / поведение (колонки выгрузки)

| Доп. статус | Поведение |
|-------------|-----------|
| ASSIGNED TO | Создан инцидент в HD; задача в БП; работа ПК |
| Done | Другой отдел завершил работу с обращением |

## Статусы в контексте «Секретарь ПК» (отдельный перечень в CSV)

В выгрузке перечислены: DRAFT, REGISTERED, IN PROGRESS, CLOSED, AUDIT, ASIGNED TO (опечатка в источнике), DONE, UNDER REVIEW, SIGNED, ARCHIVE — трактовка переходов должна быть согласована с полным текстом CSV и ТЗ.
