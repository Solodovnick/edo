## Ограничения к системе

#### 1. **Требования к совместимости**

    1. Система должна работать в операционных системах следующих версий:

        - Linux 

    1. Система должна работать на экранах с разрешением не менее 1920x1080

    2. Система должна поддерживать работу в браузерах:

        - Яндекс Браузер 20.х - 23.х

#### **2. Требования к локализации**

    - Система должна предоставлять интерфейс на русском языке

    - Cистема должна поддерживать отображение арабских цифр

    - Cистема должна функционировать по часовому поясу UTC+3

#### **3. Требования к языку программирования**

    - React 18.х + Javascript для фронт разработки

    - Springboot версия 2.7 (не ниже) Postgres версия 15.2 (не ниже) + Java для бэк разработки (не ниже) + Liberica Standart JDK

#### **4. Требования к объемам данных**

    Размер веб-страницы:

        - Веб-страница должна иметь объем включая все данные не более 20 мб

#### **5. Требования к оборудованию**

    Минимальные требования к ПК для работы с браузером:

        - Двухъядерный процессор с тактовой частотой не менее 2 ГГЦ (вычислительная мощность не менее 20 ТФлопс)

        - 2 ГБ RAM

        - 100 МБ дискового пространства

        - Доступ к сети интернет со скоростью не менее 2 Мбит/с

        - Стандартная компьютерная мышь

        - Стандартная компьютерная клавиатура

        - Монитор диагональю не менее 13 дюймов

    Минимальные требования к серверному оборудованию:

        - Процессор 8 ядерный (8 потоков), частота 3.5 ГГЦ и больше

        - Оперативная память от 32 ГБ или больше

        - Хранилище (дисковое пространство) от 300 ГБ и больше

        - Доступ к сети интернет со скоростью не менее 1 Гбит/с

    Хранилище Yandex Object Storage

        - тип накопителя SSD (read 20000), write 8000 (в режиме randrw)

        - процессор 8 ядер, частота 3.5 ГГЦ и больше

        - Оперативная память от 16 ГБ или больше

        - Пропускная способность сетевого интерфейса не менее 10 Гбит/с

#### 6. **Требования к используемым технологиям**

    - Клиентские технологии:

        - Система в части ПО, организующего взаимодействие с пользователем, должна использовать технологии:

            1. Язык разметки гипертекста не ниже HTML 5;

            2. Каскадные таблицы стилей не ниже CSS 3:

            3. Фреймворк React/Angular;

            4. Фреймворк Typescript;

            5. Вспомогательные клиентские технологии и библиотеки

            6. CryptoPro через Госключ - софт для подписания УКЭП

    - Серверные технологии:

        - Система в части ПО, организующего обработку запросов и данных, должна использовать технологии:

            1. Среда разработки и исполнения программ не младше Java 13;

            2. Спецификация Servlet API не ниже 4.0;

            3. JavaEE версии 9+ или Spring Framework версии 5+.

            НЕДОПУСТИМО использовать каркас Spring в приложении, развертываемом как артефакт JavaEE (.ear, .sar и его составляющих)

    - Технологии хранения данных:

        - Система в части ПО, организующего хранение данных, должна использовать технологии:

            1. СУБД PostgresSQL не ниже 15.версии;

            2. Файловое хранение при необходимости.

    - Технологии окружения:

        - Система в части ПО, обслуживающего процесс разработки системы, должна использовать технологии:

            1. Apache Groovy версии 2.4 и выше для обслуживающих скриптов;

            2. maven версии 3.8 и выше;

            3. NodeJS для автоматизации задач разработки клиентской части

#### **7. Требования к пользователям**

    Общие:

        - Пользователи должны читать и писать на языке интерфейса

        - Пользователи имеют активную учетную запись с доступом к модулю авторизации 

        - Пользователи должны уметь работать с системой без специального ИТ образования

    Веб-приложения:

        - Пользователи должны обладать базовыми навыками работы с ПК и Веб-браузерами

#### 8. **Требования к документированию**

    Комплект документации к программной системе должен включать:

        - Архитектурный план системы (ГОСТ Р 57100-2016)

        - SRS IEEE 830-1998

        - Руководство администратора в формате .pdf и HTML

        - Пользовательскую документацию в формате .pdf и HTML

#### 9. **Требования к интерфейсам**

    Веб-приложения:

        - Интерфейс системы должен быть рассчитан на пользователей, не имеющих специальных технических знаний и навыков в области компьютерной техники и быть легко осваиваем ими

    API:

        - Интеграция с системами должна осуществляться по REST API/SOAP или Kafka/Rabbit/Шина

#### **10. Требования к патентной чистоте (лицензированию)**

    - При разработке системы запрещено использование сторонних платных компонентов разработанных не на территории РФ
