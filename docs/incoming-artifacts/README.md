# Импорт исходной документации

**Version:** 1.0.3 | **Date:** 2026-05-03

Сюда можно складывать **полные копии** выгрузок из Telegram Desktop, ZIP архивов ТП и т.д.

Учёт материалов **на диске и по ссылкам** (до импорта и после) — в [`docs/external-sources-catalog.md`](../external-sources-catalog.md).

На текущий момент:

- Реестр UC версионирован в репозитории: **`docs/external-registry/use-case-registry.csv`** (копия из комплекта поставки).
- **Пакет «Сценарии использования» (594e282f):** распакован в [`use-cases-export-594e282f/`](use-cases-export-594e282f/README.md) — индексный MD (шаблон UC, CRUDL), CSV реестра (дубликат канонического файла по хэшу), отдельные MD по каждому сценарию. Исходный ZIP: `Сценарии+использования+594e282f-b6c1-48aa-80b1-62713d373932.zip`.
- **Полный словарь данных:** [`slovar-dannykh/`](slovar-dannykh/README.md) — импорт `Словарьданных.md` (командный + индивидуальные блоки; ER-PNG из исходника при отсутствии файла описан в тексте импорта).
- Архив «Техническое проектирование IT-решений» (ZIP) при необходимости распаковывается локально; из‑за длинных путей и кодировок автоматическая распаковка может выдавать предупреждения — допустимо держать исходный ZIP рядом с проектом и не коммитить бинарники без явной задачи.
