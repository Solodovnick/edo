# ADR-003: Tailwind CSS 4 как основа стилей

**Status:** Accepted | **Date:** 2026-05-03

---

## Контекст

Проект использует тёмную тему с кастомной цветовой палитрой. Необходимо определить подход к стилизации.

## Решение

Использовать **Tailwind CSS 4** в связке с **CSS-переменными** для дизайн-токенов. Пользовательские токены определены в `default_shadcn_theme.css`.

## Обоснование

- Утилитарный подход ускоряет разработку компонентов
- CSS-переменные для токенов позволяют централизованно менять тему
- Tailwind 4 поддерживает нативный CSS-слой без конфиг-файла (Vite-плагин)
- Совместим с shadcn/ui (который изначально разработан под Tailwind)

## Последствия

- Классы Tailwind в JSX — норма, не анти-паттерн для данного проекта
- Не использовать `style={}` для токенных значений — только `className` с Tailwind или CSS-переменные
- Адаптивные брейкпоинты Tailwind (`sm:`, `md:`, `lg:`) применяются только там, где это описано в ТЗ
