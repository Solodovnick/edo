# ADR-002: UI-стек — shadcn/ui + Radix UI + MUI

**Status:** Accepted | **Date:** 2026-05-03

---

## Контекст

Проект требует богатого набора UI-компонентов: таблицы, диалоги, формы, выпадающие меню, графики. Необходимо выбрать компонентную базу.

## Решение

Использовать **два уровня** компонентной базы:
1. **shadcn/ui + Radix UI** — для форм, диалогов, выпадающих списков, тостов (headless, полный контроль стилей)
2. **MUI (Material UI) 7** — для сложных таблиц с сортировкой и пагинацией

## Обоснование

- Radix UI обеспечивает WAI-ARIA доступность из коробки
- shadcn/ui — копируемые компоненты без скрытых зависимостей
- MUI DataGrid — лучший в классе компонент таблицы (сортировка, фильтрация, виртуализация)
- Комбинация покрывает 100% потребностей без кастомных компонентов с нуля

## Последствия

- Два источника компонентов требуют согласованности в стилях (решено через CSS-переменные)
- Вес bundle увеличен из-за MUI, но оправдан функциональностью таблиц
