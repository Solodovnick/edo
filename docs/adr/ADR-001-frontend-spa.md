# ADR-001: Frontend SPA без бэкенда (localStorage mock)

**Status:** Accepted | **Date:** 2026-05-03 | **Version:** 1.0.1

---

## Контекст

Проект EDO Bank является UI-прототипом системы управления обращениями. На текущем этапе разработки требуется демонстрация полного пользовательского пути без реального бэкенда.

## Решение

Реализовать фронтенд как Single Page Application (React + Vite) с mock-данными и хранением состояния в `localStorage`. Бэкенд не реализуется на данном этапе.

## Обоснование

- Быстрый старт: нет необходимости в настройке серверной инфраструктуры
- Полностью рабочая демонстрация UX без API
- Возможность подключения реального API в будущем через слой `services/`
- Данные в `localStorage` — изолированы от внешних сервисов

## Последствия

- Данные не сохраняются между сессиями разных браузеров
- При замене mock на реальный API — изменить только `services/*.ts`, не трогать компоненты
- Нет авторизации в MVP (роли симулируются переключением в UI)

## Связанные решения

- Для **итогового учебного проекта** (курс «Системный анализ») допускается отдельный контур **REST + PostgreSQL** по [ADR-004](ADR-004-education-demo-backend.md) и [плану разработки](../backend-development-plan.md). Режим по умолчанию в репозитории остаётся mock (**ADR-001** не отменяется).
