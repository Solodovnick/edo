## Префиксы кодов

| Префикс | Зона ответственности |
|---|---|
| **UC-REG.*** | Регистратор: создание обращений, досрочное решение, мошенничество |
| **UC-AU.*** | Аудитор: уведомления, взятие в работу, отказ от обработки, аудит, автодозвон (вне MVP) |
| **UC-AR.*** | Архив |
| **UC-RU.*** | Руководитель: просмотр, шаблоны, назначение, списки |
| **UC-ADM.*** | Административные сценарии (история обращений) |
| **UC-PRO.*** | Ответственный специалист: поиск, карточка, приём в работу, отказ, запрос в БП, решение R6 |
| **UC-INT.*** | Интеграционные (МОКИ, dadata и др.) |

## Замечания по качеству исходных данных

1. Два разных сценария помечены как **UC-AU.01** («Прочитать уведомления» и «Взять в работу») — при формализации FR назначить уникальные коды.
2. Коды **FR-PRO.*** в CSV смешаны с **UC-PRO.*** — в ТЗ использовать единый префикс (рекомендуется **UC-PRO.***).
