# Глоссарий терминов — модуль обработки обращений

**Version:** 1.0.0 | **Date:** 2026-05-03 | **Status:** Draft  
**Источник:** фрагмент «Словарь данных» (`Словарь+данных+afaff3ea….md`), CRUDL-модель из «Сценарии использования», реестр UC.

---

## Роли (упрощённо)

| Термин | Назначение |
|---|---|
| **Регистратор** | Регистрирует устные и письменные обращения, отметки по мошенничеству, досрочное решение |
| **Ответственный** | Основной исполнитель по обращению: приём в работу, запросы в БП, решение, ответ клиенту |
| **Руководитель** | Назначение ответственных, шаблоны ответов, просмотр реестров, архив (по UC) |
| **Аудитор** | Аудит после «Решено», уведомления, отказ от обработки с объяснением |
| **МОО / смежные системы** | Интеграции (МОКИ, dadata и др.) |
| **Секретарь ПК** | Участник CRUDL по части вложений и сценариев (см. матрицу прав в источнике) |

В текущем UI-прототипе репозитория часть ролей может быть **смоделирована** одним пользователем; расщепление — по мере выхода из MVP.

## Объект «Обращение» (ключевые атрибуты)

По словарю данных (фрагмент):

| Атрибут | Смысл |
|---|---|
| Идентификатор обращения | Уникальный ID |
| Номер письма | Для корреспонденции из внешнего модуля |
| Дата регистрации письма | Для письменных каналов |
| Тип заявителя | Физ. лицо / Юр. лицо |
| Тип обращения | Устное / Письменное / Регулятор |
| Категория / подкатегория | Классификация обращения |
| Приоритет | Высокий / Средний / Низкий |
| Заявитель | Ссылка на сущность заявителя |
| Содержание обращения | Текст (до 5000 символов в источнике) |
| Статус | Жизненный цикл (см. `docs/state-diagram.md`) |
| Плановая / фактическая дата исполнения | SLA и факт закрытия |
| Требует внимания | При отказе заявителя и др. |
| Вложения, ответ, запрос в БП, инцидент | Связанные сущности |

## Термины сценария (Use Case)

| Термин | Определение |
|---|---|
| **Предусловие** | Условия, при которых сценарий возможен |
| **Триггер** | Событие, запускающее сценарий |
| **Постусловие** | Результат (в т.ч. неявные изменения состояния системы) |
| **Основной поток** | Успешная последовательность шагов |
| **Расширение / альтернатива** | Ответвления (отмена, ошибки валидации) |
