# Диаграмма состояний — жизненный цикл обращения

**Version:** 1.1.0 | **Date:** 2026-05-03 | **Status:** Active  
**Источник:** `docs/functional-requirements.md`, фрагменты UC аудита (`UC-AU.*`), протоколы и реестр UC в `docs/external-registry/use-case-registry.csv`.

---

## Уточнение по статусам

В исходных текстах UC аудита встречаются формулировки **«В архиве»**, **«Пройден аудит»**, распределение из **пула аудитору** (Au-xxxx). Их нужно согласовать с единым справочником статусов в конфигурации/БД. Ниже — **объединённая модель для прототипа**: переходы можно поэтапно подтягивать к данным в `src/data/`.

---

## Диаграмма (Mermaid)

```mermaid
stateDiagram-v2
    [*] --> Черновик: создание карточки
    Черновик --> Зарегистрировано: регистратор сохраняет
    Зарегистрировано --> Назначено: назначен ответственный
    Назначено --> ВРаботе: UC-PRO.03 принять в работу
    ВРаботе --> ЗапросВБП: запрос в смежное подразделение (UC-PRO.05)
    ЗапросВБП --> ВРаботе: получен ответ от БП
    ВРаботе --> Решено: подготовлено решение / ответ (UC-PRO.06)
    Решено --> ОжидаетАудита: распределено аудитору / статус «Аудит»
    ОжидаетАудита --> АудитВРаботе: аудитор «взял в работу» (UC-AU.01)
    АудитВРаботе --> Решено: отказ аудитора → возврат в пул / перераспределение (уточняется)
    АудитВРаботе --> ПройденАудит: публикация результата аудита (UC-AU.03)
    ПройденАудит --> Закрыто: доведение ответа клиенту
    Закрыто --> Архив: истёк срок / перевод в архив (UC-AR.01)
    Архив --> [*]

    ВРаботе --> Отказано: отказ исполнителя с причиной (UC-PRO.04)
    ОжидаетАудита --> Отказано: отказ аудитора (UC-AU.02)
```

---

## Текстовая схема (кратко)

1. Регистрация → **Зарегистрировано**.  
2. Назначение → **В работе** у ответственного.  
3. При необходимости → **Запрос в БП** → возврат в работу.  
4. Готовность решения → **Решено**.  
5. Линия аудита → пул / **ожидание аудита** → **аудит в работе** → **пройден аудит** (или возврат при отказе).  
6. Закрытие → **Архив**.
