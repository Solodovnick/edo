# Excel → документация Markdown (процесс EDO Bank)

**Version:** 1.0.0 | **Date:** 2026-05-03 | **Status:** Active

Назначение: перенос таблиц из **Excel (.xlsx / .xls)** в репозиторий так, чтобы их можно было ревьюить в git и ссылаться из ТЗ.

---

## Принципы

| Ситуация | Что делать |
|----------|------------|
| **Большая таблица** (десятки/сотни строк, частые правки) | Экспорт листа в **CSV UTF-8** (или TSV) → файл в `docs/external-registry/` с говорящим именем, например `glossary-import.csv`. В `docs/*.md` — **краткая сводка**, ссылка на CSV и при необходимости выборочные строки в Markdown-таблице. |
| **Небольшая таблица** (матрица ролей, 5–20 строк) | Экспорт CSV/TSV → конвертация в Markdown-таблицу утилитой (см. ниже) → вставить или сохранить фрагмент в нужный `docs/*.md`. |
| **Исходный .xlsx** | Коммитить **только по явной задаче** (бинарник плохо диффится). Предпочтительно держать **CSV в репо** + описание в Markdown. |

---

## Экспорт из Excel

1. Открыть книгу → нужный лист.
2. **Файл → Сохранить как** → формат **CSV UTF-8 (разделители — запятые)** (*.csv)  
   или **Текст Unicode (разделители — табуляция)** для TSV.
3. Сохранить в рабочую копию, затем скопировать файл в `docs/external-registry/` (или промежуточно в `docs/incoming-artifacts/`).

Если в ячейках есть переносы строк или запятые внутри кавычек — используйте стандартный CSV UTF-8 из Excel; утилита ниже учитывает кавычки по строкам.

---

## Конвертация в таблицу Markdown

Запуск из корня репозитория (аргумент — путь к экспорту CSV или TSV из Excel):

```bash
npm run tabular-to-md -- docs/external-registry/example.csv
```

Или напрямую:

```bash
node scripts/tabular-to-markdown.mjs путь/к/файлу.csv
node scripts/tabular-to-markdown.mjs путь/к/файлу.tsv
```

Результат выводится в **stdout** — перенаправьте в файл или вставьте в документ:

```bash
node scripts/tabular-to-markdown.mjs docs/external-registry/example.csv > docs/_fragment-table.md
```

**Первая строка** файла считается заголовком столбцов. Разделитель: **табуляция**, если в первой строке есть символ табуляции; иначе **запятая** (CSV).

В ячейках символ `|` экранируется для совместимости с GitHub Flavored Markdown.

## Связь с реестром артефактов

- Новый CSV в `docs/external-registry/` — добавить строку в `docs/artifacts-catalog.md` (или отметить в существующей записи), при необходимости обновить `docs/VERSIONS.md`.
- Если таблица становится **источником требований** — зафиксировать формулировки в `docs/functional-requirements.md` (или согласовать отдельно).
