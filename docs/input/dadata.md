
# Поиск структурированных адресов через DaData [GET /address]

## Общее описание
Интеграционный метод поиска структурированных адресов через DaData предназначен для ускорения ввода адресов отправления и доставки посылок в системе курьерской службы ShipEasyGA, устранения возможных ошибок ввода и необходимости ручной проверки адресов сотрудниками после оформления заказа услуги. 

Метод будет использоваться пользователями веб- и мобильных приложений.

## API-документация Внешней системы
🔗 https://dadata.ru/api/suggest/address/

**Ограничения Внешней системы:**
* Для иностранных адресов заполнен ограниченный набор полей, вне зависимости от тарифного плана: почтовый индекс, страна, регион, населенный пункт, часовой пояс и геокоординаты. Прочие поля - пустые.
* Длина запроса (параметр `query`) - не более 300 символов.
* Максимальная частота запросов - 30 в секунду с одного IP-адреса.
* Количество запросов в день - в соответствии с тарифным планом.
* Максимальная частота создания новых соединений - 60 в минуту с одного IP-адреса.

## Сценарий работы (Use Case)

**Системы:**
* Клиент (web/мобильное приложение GA)
* Backend ShipEasyGA (REST API)
* DaData

**Предусловие:** Авторизация пользователя ShipEasyGA в ЛК.

**Входные данные:**
* Строка адреса в свободном формате: `text`
* Тип адреса - отправления или доставки: `type`
* `Bearer Token`

**Алгоритм работы:**
Клиент вызвал API-метод и передал на вход строку адреса.

1. **Выполняется проверка полученной строки адреса `text`:**
   * Если в строке адреса `text` получено менее 10 символов, то вернуть пустой результат.
   * Если в строке адреса `text` получено более 200 символов, то вернуть пустой результат. Не возвращаем ошибку, т.к. предлагаем пользователю вводить адрес вручную и не нужно показывать ошибку на UI.
   * Иначе - далее.

2. **Backend ShipEasyGA вызывает внешнюю систему DaData**, чтобы найти адреса соответствующие полученной строке `text`:
   * `POST http://suggestions.dadata.ru/suggestions/api/4_1/rs/suggest/address`
   * с телом запроса, содержащим адрес: `{"query": "сочи, севастопольская, 123"}` 
   *(P.S. для больших body тут просто отсылка на маппинг, который описывается после сценария)*

3. **API DaData возвращает ответ** в виде массива объектов с адресами `suggestions`.

4. **Backend ShipEasyGA выбирает из ответа только нужные данные** по каждому адресу и формирует JSON для ответа в соответствии с требованиями к маппингу данных, описанному ниже.
   * Возвращать только первые 5 рекомендуемых DaData адресов. Остальные значения в массиве, полученном от DaData, игнорировать.
   * *Также Backend сверяет координаты первого по порядку предложенного адреса с координатами пунктов курьерской службы. Если в радиусе 5 км от адреса они есть, то такие пункты также возвращаются в ответе отдельным списком, до 3-х пунктов. (Это лучше вынести в отдельный метод с подсказками по ближайшим адресам курьерской службы, чтобы не усложнять описываемый метод).*

5. **Сформированный JSON возвращается в ответ** на клиента, который вызвал метод.

### Альтернативные сценарии:

* **1А. Если строка адреса `text` не передана или получено пустое значение `text=''`**:
  * Возвращать список из 5 последних адресов клиента для пунктов отправления или прибытия, в зависимости от параметра `type` (`pickup` - отправления, `delivery` - доставки).
  * Для поиска адресов в БД используется таблица БД `address`. Выбираются все записи для авторизованного пользователя по `client_id` (определяем `client_id` через `Bearer Token` авторизации).
  * Из найденных записей выбираются только те, которые соответствуют переданному типу `type`, если он передан (поле в БД `address.is_pickup`): 
    * `is_pickup=true` - соответствует `pickup`
    * иначе - `delivery`.
  * Найденные записи адресов сортируются по дате создания `created_at` в БД. Берутся 5 самых новых (или сколько найдено) и возвращаются самые актуальные. В результате может быть возвращено от 0 до 5 записей.

* **1B. Ошибка авторизации запроса.** 
  * Передан недействительный Bearer-токен пользователя. Вернуть ошибку HTTP-401.

* **3А. В случае получения любой ошибки от DaData API** (т.е. если получен не HTTP-200 код ответа): 
  * Вернуть пустой результат на клиента.

* **3B. От DaData API получен пустой результат** в `suggestions`:
  * Вернуть пустой результат на клиента.

* **4A. Ошибка парсинга JSON** (перестал соответствовать требованиям к маппингу):
  * Вернуть пустой результат на клиента.
  * Записать ошибку парсинга запроса в лог системы, включающую запрос и ответ системы DaData.

## Техническая реализация метода

### Запрос
* **Тип метода:** `GET`
* **URL:** `https://shipeasyga.com/api/v1/address`

**Headers:**
* `Content-Type: application/JSON`
* Авторизация: `Bearer Token пользователя`

**Query-параметры:**
* `text`
* `type`

**Пример URL запроса:**
```http
GET /address?text='строка адреса'&type=pickup
```

**Тело запроса:** Нет

### Ответ

**Успешный ответ - есть результаты:**
*Пример для строки: `владимир ивановская подгорная 2 к3`*
* **Код ответа:** `HTTP-200`
* **Тело ответа (JSON):**

```json
{
  "results": [
    {
      "fullAddress": "Владимирская обл, г Владимир, ул Ивановская Подгорная, д 2 к 3",
      "postalCode": null,
      "country": "Россия",
      "countryIsoCode": "RU",
      "region": "Владимирская обл",
      "regionType": "область",
      "regionIsoCode": "RU-VLA",
      "city": "Владимир",
      "cityType": "город",
      "street": "Ивановская Подгорная",
      "streetType": "улица",
      "house": "12А",
      "houseType": "дом",
      "block": "3",
      "blockType": "корпус",
      "apartment": null,
      "apartmentType": null,
      "latitude": "55.686589",
      "longitude": "37.600971"
    }
  ]
}
```

**Успешный ответ - результаты не найдены:**
* **Код ответа:** `HTTP-200`
* **Тело ответа (JSON):**
```json
{
  "results": []
}
```

## Требования к обработке ошибок

**Ошибка авторизации**
* **Код:** `HTTP-401`
* **Тело ответа:**
```json
{
  "code": 401,
  "message": "Пользователь не авторизован"
}
```

## Маппинг данных

### Запрос

| Параметр | В API ShipEasyGA | Обяз. в API ShipEasyGA | В API DaData | Комментарий (требования к валидации, алгоритмам) |
| :--- | :--- | :--- | :--- | :--- |
| Строка для поиска адреса | `text` (query-параметр) | Нет | `query` (body) | Строка должна содержать не менее 10 символов. Если параметр не передан, то возвращать список из 5 последних адресов клиента для пунктов отправления или прибытия, в зависимости от query-параметра `type`. |
