## 1. Дизайн-концепция

**Стиль:** Ретро-футуристический (Winamp-inspired) — тёмная тема, неоновые акценты, панели с чёткими границами.

**Принципы:**
- Компактность — максимум информации на экране без прокрутки
- Трёхколоночный layout для карточек обращений (левый сайдбар / центр / правый сайдбар)
- Цветовая семантика статусов (зелёный — ОК, жёлтый — предупреждение, красный — нарушение)

## 4. Компоненты

### 4.1 Таблица обращений
- Строки с hover-состоянием (подсветка)
- Клик по строке → открытие карточки
- Колонка «Статус» — цветные бейджи
- Колонка «Дедлайн» — таймер обратного отсчёта (красный при нарушении)

### 4.2 Карточка обращения (3-колоночный layout)
```
┌─────────────┬──────────────────────┬─────────────┐
│ Левый       │ Центр                │ Правый      │
│ сайдбар     │ (основной контент)   │ сайдбар     │
│             │                      │             │
│ Заявитель   │ Исходное обращение   │ Управление  │
│ История     │ Текст решения        │ Комментарии │
│ Дедлайн     │ Поля ввода           │ Вложения    │
│ Автодозвон  │ Кнопки действий      │             │
└─────────────┴──────────────────────┴─────────────┘
```

### 4.3 CRM-индикатор
- **Серый** — поле пустое («Введите имя»)
- **Зелёный** — клиент найден (данные заполнены)
- **Жёлтый** — клиент не найден в базе

### 4.4 Боковая навигация (Sidebar)
- Иконки + текстовые метки
- Активный раздел — акцентный цвет
- Свёртываемая (collapsed mode)

### 4.5 KPI-карточки Dashboard
- Иконка + числовое значение + тренд (↑/↓)
- 4 карточки в ряд: объём, FCR, время обработки, нарушения SLA

## 6. Состояния UI

| Состояние | Отображение |
|---|---|
| Загрузка | Skeleton-placeholder или спиннер |
| Пустой список | Текст «Нет обращений» + иллюстрация |
| Ошибка | Toast-уведомление (sonner) |
| Успешное действие | Toast зелёный + обновление данных |
| SLA нарушен | Красная подсветка строки + иконка предупреждения |
