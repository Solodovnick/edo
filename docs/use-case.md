## Ключевые use cases (черновик)

### UC-01 Регистрация обращения
**Актор:** Ответственный специалист  
**Предусловие:** Пользователь авторизован  
**Основной поток:**
1. Открывает форму «Новое обращение»
2. Выбирает категорию заявителя (физлицо / юрлицо / регулятор)
3. Вводит имя заявителя → система ищет в CRM
4. При нахождении — автозаполнение данных
5. Вводит текст обращения, выбирает тип (письменное / устное)
6. Подтверждает → обращение создаётся со статусом «Новое»

### UC-02 Обработка обращения (Кабинет ответственного)
**Актор:** Ответственный специалист  
**Предусловие:** Обращение назначено специалисту  
**Основной поток:**
1. Открывает «Мои обращения»
2. Применяет фильтр / поиск
3. Кликает на строку → открывается карточка
4. Знакомится с текстом обращения
5. Вводит текст решения
6. Сохраняет → статус → «Решено»

### UC-03 Аудит решения
**Актор:** Аудитор  
**Предусловие:** Обращение имеет статус «Решено»  
**Основной поток:**
1. Открывает кабинет аудитора
2. Выбирает обращение из таблицы
3. Читает исходное обращение и решение специалиста
4. Выбирает шаблон аудита
5. Вводит комментарий аудитора
6. Публикует результат → статус → «Проверено»

### UC-04 Мониторинг SLA
**Актор:** Администратор  
**Предусловие:** В системе есть активные обращения  
**Основной поток:**
1. Открывает страницу SLA-мониторинга
2. Видит таблицу с таймерами и статусами дедлайнов
3. Фильтрует: нарушенные / критические
4. При необходимости — переназначает обращение
