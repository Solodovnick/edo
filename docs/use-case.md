## UC-REG.* — Регистратор

### UC-REG.01 / UC-INT.01 Создать устное обращение
**Актор:** Регистратор (интеграционный вариант — ответственный).  
**Поток (обобщённо):** открыть форму устного обращения → заполнить обязательные поля (тип заявителя, содержание, категория при наличии) → сохранить → статус не ниже «Зарегистрировано».

### UC-REG.02 Создать письменное обращение
**Актор:** Регистратор.  
Учитываются номер/дата регистрации письма из процесса корреспонденции (по словарю данных).

### UC-REG.03 Решить обращение досрочно
**Актор:** Регистратор.  
Инициирует завершение без полного цикла при допустимых бизнес-правилах.

### UC-REG.04 Сообщить о мошеннике
**Актор:** Регистратор.  
Отдельный сценарий фиксации инцидента (связь с сущностью «Инцидент» в модели данных заказчика).

## UC-PRO.* — Ответственный специалист

### UC-PRO.01 Найти обращение
Поиск по номеру, атрибутам заявителя (см. FR-04.4).

### UC-PRO.02 Просмотреть обращение
Карточка обращения (трёхколоночный layout).

### UC-PRO.03 Принять обращение в работу
Переход в статус «В работе» исполнителем.

### UC-PRO.04 Отказаться от обращения
Форма с **объяснительной причиной**, уведомление руководителю (по шаблону исходного UC ответственного; для аудита аналогично UC-AU.02).

### UC-PRO.05 Запросить информацию в смежном подразделении
Запрос в БП, ожидание ответа, возврат в работу.

### UC-PRO.06 R6 — принять решение, подготовить и довести ответ клиенту
Ключевой сценарий закрытия линии ответственного до передачи на аудит/архив.

### UC-PRO.00 Связаться по негативной обратной связи
Отдельный контакт при негативе от клиента.

---

## UC-AU.* — Аудитор

### UC-AU.01 Взять обращение в работу (аудит)
**Предусловия:** обращение в статусе «Решено», назначение аудитору / статус линии «Аудит».  
**Поток:** уведомление → реестр «мои обращения» → перевод в работу у аудитора.

### UC-AU.02 Отказ от обработки аудитором
Диалог с причиной, уведомление руководителю.

### UC-AU.03 Провести аудит
Шаблон аудита, оценки, публикация; переход к архиву или статусу «пройден аудит» (см. `state-diagram.md`).

### UC-AU.04 Автодозвон роботом
**Вне MVP** прототипа (пометка в реестре).

## UC-RU.* / UC-AR.* / UC-ADM.*

- **UC-RU-04** — просмотр списков обращений для руководителя и совместно с другими ролями.
- **UC-RU-03** — назначение ответственного.
- **UC-RU-02 / UC-RU-05** — шаблоны ответов (создание и редактирование).
- **UC-AR.01** — работа с архивом.
- **UC-ADM.01** — отображение истории обращений для руководителя и аудитора.
