## BR-01 Контекст и проблема

**Домен:** Банковский сектор — обработка обращений клиентов (жалобы, запросы, регуляторные обращения).

**Проблема:**
- Ручная обработка обращений в разрозненных таблицах и почте приводит к нарушениям SLA
- Отсутствие единой истории клиента (CRM) замедляет идентификацию заявителя
- Нет инструмента аудита качества ответов специалистов
- Аналитика по KPI ведётся вручную

**Решение:** Единая система управления обращениями с кабинетами специалистов, аудитом, CRM и аналитикой.

## BR-02 Бизнес-цели

| ID | Цель | Метрика успеха |
|---|---|---|
| BG-01 | Снизить время обработки обращения | Среднее время обработки ↓ 30% |
| BG-02 | Обеспечить соблюдение SLA | Доля нарушений SLA < 5% |
| BG-03 | Повысить качество ответов | FCR (First Contact Resolution) ≥ 80% |
| BG-04 | Обеспечить прозрачность аудита | 100% обращений проходят аудиторскую проверку |
| BG-05 | Снизить трудозатраты на аналитику | Отчёты формируются автоматически |

## BR-03 Категории заявителей и типы обращений

| Категория | Особенности |
|---|---|
| Физическое лицо | ФИО, телефон, дата рождения, адрес, VIP-статус |
| Юридическое лицо | Наименование ООО, ИНН, адрес |
| Регулятор (ЦБ РФ) | Обязательный ответ в установленные сроки, поле ЦБС |

| Тип обращения | Примечание |
|---|---|
| Письменное | Корреспонденция, вложения |
| Устное | Телефон/офис |
| Регуляторное | Отдельный контур требований и SLA (ЦБ РФ и др.) |

## BR-03а Акторы процесса (по реестру UC)

В дополнение к упрощённой модели UI-прототипа закрепляются акторы из проектной документации:

| Роль | Функции (обобщённо) |
|---|---|
| **Регистратор** | UC-REG.* — регистрация каналов, мошенничество, досрочное решение |
| **Ответственный специалист** | UC-PRO.* — исполнение, запросы в БП, R6 |
| **Руководитель** | UC-RU.*, UC-AR.* — назначения, шаблоны, архив |
| **Аудитор** | UC-AU.* — линия аудита после решения |
| **МОО / интеграции** | UC-INT.* — внешние справочники |

## BR-04 Роли пользователей (матрица MVP vs полная модель)

| Роль | Описание | Права (целевые) |
|---|---|---|
| Регистратор | Ввод обращений в систему | По матрице CRUDL для обращений на регистрации |
| Ответственный специалист | Исполнение назначенных обращений | Работа с карточкой, запросы в БП, решение, отказ с причиной |
| Аудитор | Контроль качества после решения | Линия UC-AU.*, шаблоны аудита |
| Руководитель | Распределение и контроль | Назначение ответственных, шаблоны ответов, архивные выборки |
| Администратор (системный) | Настройки экземпляра | Пользователи, роли, SLA — см. FR-10 |

В прототипе допускается **ограниченный набор ролей в UI** до полной реализации матрицы доступа.
