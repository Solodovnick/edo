## 🎯 Три состояния индикатора

### 1. **"Введите имя"** - Пустое поле (серый)

**Условие:**
- ФИО заявителя не заполнено (`applicantName.trim() === ''`)
- ИЛИ Наименование организации пустое (`organizationName.trim() === ''`)

**Визуальный дизайн:**
```
┌─────────────────┐
│ • Введите имя   │ ← Серый градиент + серая точка
└─────────────────┘
```

**Стиль:**
- Градиент: `from-[#e0e0e0] to-[#f5f5f5]`
- Цвет текста: `text-gray-500`
- Точка: `bg-gray-400` с серым glow
- Тень: Слабая, нейтральная

**Когда показывается:**
- При первой загрузке страницы (пустая форма)
- После сброса формы
- Когда пользователь очистил поле ФИО

---

### 2. **"✓ Клиент"** - Найден в базе (зелёный)

**Условие:**
- ФИО введено И найдено в тестовой базе `TEST_CLIENTS`
- Совпадение по точному имени (case-insensitive)

**Визуальный дизайн:**
```
┌─────────────────┐
│ • ✓ Клиент      │ ← Зелёный градиент + зелёная точка
└─────────────────┘
```

**Стиль:**
- Градиент: `from-[#a8edea] to-[#96fbc4]` (бирюзово-зелёный)
- Цвет текста: `text-gray-800`
- Точка: `bg-green-500` с ярким зелёным glow
- Тень: Яркая, цветная

**Автозаполнение:**
При обнаружении клиента автоматически заполняются:
- Дата рождения
- Телефон
- Адрес
- Персональный менеджер
- VIP статус
- История обращений

**Пример:**
```
Пользователь вводит: "Райан Гослинг"
→ Индикатор: "✓ Клиент" (зелёный)
→ Автозаполнение:
   - Телефон: +7 (999) 001-00-01
   - Дата рождения: 12/11/1980
   - Адрес: г. Лос-Анджелес...
   - ПМ: Сидоров П.П.
   - VIP: Да
```

---

### 3. **"○ Не клиент"** - Не найден в базе (оранжевый)

**Условие:**
- ФИО введено НО не найдено в тестовой базе
- Имя не совпадает ни с одним из `TEST_CLIENTS`

**Визуальный дизайн:**
```
┌─────────────────┐
│ • ○ Не клиент   │ ← Розовый градиент + оранжевая точка
└─────────────────┘
```

**Стиль:**
- Градиент: `from-[#ffd6e8] to-[#ffc0da]` (розовый)
- Цвет текста: `text-gray-800`
- Точка: `bg-orange-500` с оранжевым glow
- Тень: Средняя, цветная

**Поведение:**
- Поля остаются пустыми
- Пользователь заполняет данные вручную
- Новый клиент (не в CRM)

**Пример:**
```
Пользователь вводит: "Петров Петр Петрович"
→ Индикатор: "○ Не клиент" (оранжевый)
→ Автозаполнения нет
→ Пользователь вводит данные сам
```

## 🔍 Логика поиска

### Функция проверки клиента:

```typescript
const checkClientByName = (name: string): 'Клиент' | 'Неклиент' | 'Пусто' => {
  // 1. Проверка на пустоту
  if (!name || name.trim() === '') {
    return 'Пусто';
  }
  
  // 2. Нормализация для поиска
  const normalizedInput = name.trim().toLowerCase();
  
  // 3. Поиск в базе TEST_CLIENTS
  const foundClient = TEST_CLIENTS.find(client => 
    client.name.toLowerCase() === normalizedInput
  );
  
  // 4. Результат
  return foundClient ? 'Клиент' : 'Неклиент';
};
```

### Особенности:

**Case-insensitive поиск:**
- "Райан Гослинг" = "райан гослинг" = "РАЙАН ГОСЛИНГ" ✅

**Trimming:**
- "  Райан Гослинг  " = "Райан Гослинг" ✅

**Точное совпадение:**
- "Райан Гослинг" ✅ Найдено
- "Райан" ❌ Не найдено (неполное имя)
- "Гослинг" ❌ Не найдено (только фамилия)

## 🔄 Автозаполнение при обнаружении клиента

### Алгоритм:

```typescript
onChange={(e) => {
  const newName = e.target.value;
  setApplicantName(newName);
  
  // 1. Проверка статуса
  const status = checkClientByName(newName);
  
  // 2. Если клиент найден
  if (status === 'Клиент') {
    setClientType('Клиент');
    
    // 3. Поиск клиента в базе
    const client = TEST_CLIENTS.find(c => 
      c.name.toLowerCase() === newName.trim().toLowerCase()
    );
    
    // 4. Автозаполнение всех полей
    if (client) {
      setBirthDate(client.birthDate);
      setPhone(client.phone);
      setAddress(client.address);
      setPersonalManager(client.personalManager);
      setHasPersonalManager(true);
      setIsVIP(client.isVIP);
      setAppealHistory(client.appealHistory);
    }
  }
  
  // 5. Если клиент не найден
  else if (status === 'Неклиент') {
    setClientType('Неклиент');
    // Поля остаются как есть (не очищаем)
  }
}}
```
