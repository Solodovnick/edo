# 🧪 Инструкция по тестированию LocalStorage интеграции

## Быстрый старт

### 1. Создать обращение

1. Откройте приложение
2. Перейдите на вкладку **"Регистратор"**
3. Нажмите кнопку **"Создать обращение"** (фиолетовая кнопка)
4. Заполните форму:

```
Тип заявителя: Физ лицо
Тип обращения: Устное
ФИО: Тестов Тест Тестович
Телефон: +7 999 111 22 33
Ответственный: Расул Рамазанов
Дедлайн: выберите завтрашнюю дату
Содержание: Клиент сообщает о проблеме со входом в мобильное приложение
```

5. Нажмите **"Сохранить обращение"**
6. ✅ Увидите уведомление: "Обращение №XXXXXXX успешно зарегистрировано!"

---

### 2. Проверить в кабинете ответственного

1. Перейдите на вкладку **"Ответственный"**
2. Выберите фильтр **"Мои обращения"**
3. ✅ Ваше обращение должно быть в списке (последнее)
4. Обратите внимание:
   - Номер обращения
   - Статус "В работе"
   - Дата регистрации (сегодня)
   - Бейдж "МОЁ"

---

### 3. Открыть и заполнить карточку

1. Кликните на созданное обращение
2. ✅ Откроется карточка обработки с 3 колонками (или 3 вкладками на мобильном)
3. ✅ Поле "Исходное обращение" уже заполнено вашим текстом
4. Добавьте текст в поле **"Текст решения"**:

```
Выполнена диагностика учетной записи.
Проблема решена путем сброса кэша приложения.
Клиент успешно авторизовался.
```

5. Добавьте **"Форма ответа клиенту"**:

```
Уважаемый Тест Тестович!

Проблема со входом в мобильное приложение решена.

Для корректной работы рекомендуем:
1. Обновить приложение до последней версии
2. Проверить стабильность интернет-соединения

С уважением,
Служба поддержки
```

6. Подождите 2 секунды (автосохранение)
7. Нажмите кнопку "Назад" (стрелка влево)

---

### 4. Проверить сохранность данных

1. Найдите ваше обращение в списке снова
2. Откройте его
3. ✅ Все три поля должны быть заполнены тем, что вы ввели
4. Закройте вкладку браузера
5. Откройте приложение заново
6. Перейдите в "Ответственный"
7. ✅ Ваше обращение всё ещё на месте со всеми данными

---

### 5. Протестировать фильтры

В кабинете ответственного попробуйте:

**Фильтр по типу:**
- Нажмите "Физ лицо" ✅ Ваше обращение видно
- Нажмите "Юр лицо" ✅ Ваше обращение скрыто
- Нажмите "Все обращения" ✅ Всё видно

**Поиск:**
- Введите номер вашего обращения ✅ Найдено
- Введите "Тестов" ✅ Найдено
- Введите "мобильное приложение" ✅ Найдено

## 🔍 Проверка через DevTools

### Посмотреть данные в LocalStorage:

1. Откройте DevTools (F12)
2. Перейдите на вкладку **Application** (Chrome) или **Storage** (Firefox)
3. Слева выберите **Local Storage** → ваш домен
4. Найдите ключ `edo_appeals`
5. ✅ Увидите JSON массив с вашими обращениями

### Посмотреть структуру данных:

Скопируйте значение `edo_appeals` и вставьте в JSON formatter:
```json
[
  {
    "id": "123456789",
    "regDate": "03/03/26",
    "applicantName": "Тестов Тест Тестович",
    "content": "Клиент сообщает о проблеме со входом...",
    "solution": "Выполнена диагностика...",
    "response": "Уважаемый Тест Тестович!...",
    ...
  }
]
```

## 🧹 Очистка тестовых данных

### Вариант 1: Через консоль

```javascript
localStorage.removeItem('edo_appeals');
location.reload();
```

### Вариант 2: Через код

В любом компоненте добавьте временную кнопку:
```tsx
<button onClick={() => {
  localStorage.removeItem('edo_appeals');
  alert('Данные очищены!');
  window.location.reload();
}}>
  Очистить LocalStorage
</button>
```

## 🐛 Возможные проблемы

### Обращение не появляется в списке

**Причина:** Фильтр "Мои обращения" активен, но `isMine = false`

**Решение:** 
1. Переключите на "Все обращения"
2. Или измените ответственного на "Расул Рамазанов" или "Александр Солодовников"

### Данные не сохраняются

**Причина:** LocalStorage отключён или заполнен

**Проверка:**
```javascript
// В консоли
try {
  localStorage.setItem('test', 'test');
  localStorage.removeItem('test');
  console.log('✅ LocalStorage работает');
} catch (e) {
  console.error('❌ LocalStorage недоступен', e);
}
```

### Автосохранение не работает

**Причина:** Не ждёте 1 секунду после редактирования

**Решение:** Подождите минимум 1 секунду перед закрытием карточки
