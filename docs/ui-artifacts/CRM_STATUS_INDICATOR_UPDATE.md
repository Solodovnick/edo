# CRM Status Indicator - Обновление UI

## ✅ Изменение выполнено: 01.03.2026

### Проблема (AS IS):
Поле "Клиент/Неклиент" было представлено в виде **кнопок выбора**, что вводило в заблуждение:
- Выглядело как ручной выбор пользователя
- Неочевидно, что это результат автоматического поиска в CRM
- Две кнопки занимали много места

### Решение (TO BE):
Заменено на **индикатор статуса CRM-поиска** - read-only badge:
- ✓ **"✓ В CRM"** - клиент найден в CRM (зелёный градиент)
- ✓ **"○ Не найден"** - клиент не найден (оранжевый градиент)

---

## 🎨 Дизайн индикатора

### Визуальные характеристики:

**Клиент найден:**
```tsx
- Градиент: from-[#a8edea] to-[#96fbc4] (бирюзово-зелёный)
- Светящаяся точка: green-500 с glow эффектом
- Текст: "✓ В CRM"
- Claymorphism: inset shadows + drop shadow
```

**Клиент НЕ найден:**
```tsx

- Светящаяся точка: orange-500 с glow эффектом
- Текст: "○ Не найден"
- Claymorphism: inset shadows + drop shadow
```

### Размеры:
- **На форме**: `px-3 md:px-4 py-1.5 md:py-2 text-xs md:text-sm`
- **В превью**: `px-2.5 py-1 text-xs` (более компактный)

## 🔄 Логика работы индикатора

### Автоматическое обновление:

**1. При вводе телефона (onBlur):**
```typescript
handlePhoneBlur() {
  // CRM API search...
  
  if (foundInCRM) {
    setClientType('Клиент');      // ✓ В CRM (зелёный)
    // Auto-fill: name, address, manager, etc.
  } else {
    setClientType('Неклиент');    // ○ Не найден (оранжевый)
    // Clear all fields
  }
}
```

**2. При смене типа заявителя:**
- Индикатор сбрасывается
- Показывает "○ Не найден" до следующего поиска

**3. При автозаполнении (Sparkles кнопка):**
- Автоматически устанавливает "✓ В CRM"

## ➕ Добавленный код

### Индикатор на форме (2 места):

**Для Физлиц (после input ФИО):**
```tsx
{/* CRM Status Indicator */}
<div 
  className={`flex items-center gap-2 px-3 md:px-4 py-1.5 md:py-2 rounded-full font-bold text-xs md:text-sm whitespace-nowrap ${
    clientType === 'Клиент' 
      ? 'bg-gradient-to-br from-[#a8edea] to-[#96fbc4] text-gray-800' 
      : 'bg-gradient-to-br from-[#ffd6e8] to-[#ffc0da] text-gray-800'
  }`}
  style={{ /* Claymorphism shadows */ }}
>
  <div className={`w-2 h-2 rounded-full ${
    clientType === 'Клиент' ? 'bg-green-500' : 'bg-orange-500'
  }`} 
       style={{ /* Glow effect */ }}
  />
  {clientType === 'Клиент' ? '✓ В CRM' : '○ Не найден'}
</div>
```

**Для Юрлиц (после input Наименование):**
- Идентичный индикатор

```tsx
<div className="flex items-center justify-between mb-2">

  <div className={...}>

    {clientType === 'Клиент' ? '✓ В CRM' : '○ Не найден'}
```

## 🎯 Use Cases

### UC-1: Клиент найден в CRM
```
1. Пользователь вводит телефон: +7 (999) 123-45-67
2. onBlur → автопоиск в CRM
3. Найдено! → Индикатор: "✓ В CRM" (зелёный)
4. Автозаполнение: ФИО, адрес, ПМ
5. Preview обновляется с badge "✓ В CRM"
```

### UC-2: Клиент НЕ найден в CRM
```
1. Пользователь вводит телефон: +7 (900) 000-00-00
2. onBlur → автопоиск в CRM
3. Не найдено → Индикатор: "○ Не найден" (оранжевый)
4. Поля остаются пустыми
5. Toast: "⚠ Клиент не найден в CRM. Заполните данные вручную"
```

### UC-3: Автозаполнение (Sparkles)
```
1. Пользователь нажимает "Автозаполнение"
2. Все поля заполняются тестовыми данными
3. Индикатор: "✓ В CRM" (зелёный)
4. Preview показывает badge "✓ В CRM"
```
