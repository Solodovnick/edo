## 🎨 Дизайн Live Preview:

### Цветовая схема:
- **Фон панели**: Градиент от #fff4e8 до #ffeef8
- **Левая граница**: 4px solid #667eea (синий банка)
- **Карточка**: Белая с Claymorphism shadows

### Блоки в Preview:

#### 1. **Номер обращения**
- Если зарегистрировано: Большой gradient текст с номером
- Если нет: Серый плейсхолдер "Обращение не зарегистрировано"

#### 2. **Тип заявителя + Тип обращения**
- Grid 2 колонки
- Иконки: User (физлицо) / Building2 (юрлицо)
- Градиентные фоны с Claymorphism

#### 3. **Заявитель**
- ФИО или Название организации
- Телефон с иконкой Phone
- Белая карточка с border

#### 4. **VIP Статус и ПМ**
- Показывается только если `hasPersonalManager = true`
- Иконка Crown + badge VIP (если isVIP)
- Розово-голубой градиент
- ФИО персонального менеджера

#### 5. **Категория**
- Показывается если заполнена
- Название категории + подкатегория
- Специальный badge для критичных (Эквайринг → Сбой): "⚡ SLA 1 день"

#### 6. **Приоритет**
- Динамический градиент по уровню:
  - **Высокий**: #f093fb → #f5576c (красно-розовый, белый текст)
  - **Средний**: #ffecd2 → #fcb69f (оранжевый)
  - **Низкий**: #a8edea → #fed6e3 (бирюза-розовый)

#### 7. **SLA Даты**
- Grid 2 колонки
- **Регистрация**: Стандартный градиент + иконка Calendar
- **Плановая дата**: Цветная индикация по SLA:
  - Красный (#DD0000): < 2 дня
  - Желтый (#FFAA00): < 5 дней
  - Зеленый (#00AA44): ≥ 5 дней
- Иконка Clock с цветом SLA

#### 8. **Тема обращения**
- Показывается если заполнена
- Серый фон, текст

#### 9. **Содержание обращения**
- Показывается если заполнено
- Белая карточка с border
- Max-height: 32 (overflow scroll)
- Inset shadow

#### 10. **FCR Статус**
- Показывается если `fcrChecked = true`
- Зеленый border + фон
- Иконка CheckCircle
- Текст: "FCR - Решено на первой линии"

#### 11. **Кнопка регистрации (превью)**
- Disabled версия кнопки из формы
- Opacity: 70%
- Подсказка: "Используйте кнопку в форме"

## 📱 Responsive Behavior:

### Mobile/Tablet (< 1280px):
```css
grid-cols-1
```
- Показывается **только левая панель** (форма)
- Правая панель **скрыта** (`hidden xl:flex`)
- Полная ширина экрана для формы

### Desktop (≥ 1280px):
```css
grid-cols-2
```
- Обе панели видны
- 50/50 split screen
- Независимый overflow для каждой панели

## 🎯 UX Особенности:

### 1. **Условное отображение**
Блоки показываются только если заполнены соответствующие поля:
- VIP блок → если `hasPersonalManager && personalManager`
- Категория → если `category`
- Тема → если `appealTheme`
- Содержание → если `appealContent`
- FCR → если `fcrChecked`

### 2. **Визуальная обратная связь**
- Пустые поля → серый плейсхолдер "Не указано"
- Критичные категории → красный badge "⚡ SLA 1 день"
- VIP клиенты → золотая корона + желтый badge
- SLA нарушение → красный цвет даты

### 3. **Claymorphism стиль**
Все элементы preview используют:
- Inset shadows для объёмности
- Rounded углы (16px, 18px, 20px, 24px)
- Градиентные фоны
- Мягкие тени

### 4. **Scroll поведение**
- Обе панели независимо скроллятся
- Preview не перекрывает форму
- Footer всегда внизу экрана

## ✨ Особенности реализации:

### 1. **Динамический цвет SLA**
Функция `getSLAColor()` используется для:
- Фона плановой даты
- Цвета текста
- Цвета иконки Clock
- Свечения (box-shadow)

### 2. **Условная стилизация приоритета**
Inline стили с тернарными операторами:
```tsx
background: priority === 'Высокий' 
  ? 'linear-gradient(135deg, #f093fb 0%, #f5576c 100%)'
  : priority === 'Средний'
  ? 'linear-gradient(135deg, #ffecd2 0%, #fcb69f 100%)'
  : 'linear-gradient(135deg, #a8edea 0%, #fed6e3 100%)'
```

### 3. **Отзывчивая типография**
Preview использует меньшие размеры текста для компактности:
- Заголовки: text-xs
- Основной текст: text-sm
- Номер обращения: text-3xl
