# Bug Fix: Year 4026 → 2026

## ✅ Исправлено: 01.03.2026

### 🐛 Проблема:

На странице "Регистратор" в поле **"Плановая дата"** отображался год **4026** вместо **2026**.

**Скриншот:**
```
Плановая дата: 08/03/4026  ❌
```

**Ожидалось:**
```
Плановая дата: 08/03/2026  ✅
```

---

## 🔍 Причина бага

### Логика расчёта даты:

**Файл:** `/src/app/components/lifecycle/RegistrationPageEnhanced.tsx:122`

**Было:**
```tsx
const [day, month, year] = registrationDate.split('/').map(Number);
const regDate = new Date(2000 + year, month - 1, day);
```

### Проблема:

1. `toLocaleDateString('ru-RU')` возвращает **полный год** в формате `dd.MM.yyyy`:
   - Пример: `"01.03.2026"` → после замены `/` → `"01/03/2026"`

2. При парсинге:
   - `year = 2026` (полный год)

3. При создании `Date`:
   - `2000 + year` = `2000 + 2026` = **4026** ❌

### Корень проблемы:

Код предполагал, что год всегда будет в **коротком формате** (26), но `toLocaleDateString('ru-RU')` возвращает **полный год** (2026).

---

## ✅ Решение

### Исправленный код:

```tsx
const [day, month, year] = registrationDate.split('/').map(Number);
// Исправление: если год уже полный (например, 2026), не добавляем 2000
const fullYear = year < 100 ? 2000 + year : year;
const regDate = new Date(fullYear, month - 1, day);
```

### Логика:

- **Если `year < 100`** (например, `26`) → добавляем `2000` → `2026` ✅
- **Если `year >= 100`** (например, `2026`) → используем как есть → `2026` ✅
