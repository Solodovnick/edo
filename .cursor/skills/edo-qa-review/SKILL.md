# EDO Bank — QA / Ревью интерфейса

## Когда применять

- Проверка компонента или экрана на соответствие `docs/functional-requirements.md`
- Составление отчёта о расхождениях перед релизом
- Проверка acceptance criteria user story
- Аудит доступности (a11y) интерфейса

## Инструкции

1. **Читай ТЗ** перед проверкой: `docs/functional-requirements.md` и `docs/ui-ux-brief.md`.
2. **Формат отчёта:**
   ```
   ## QA-отчёт: [название компонента]
   
   | # | FR | Проверка | Статус | Примечание |
   |---|---|---|---|---|
   | 1 | FR-04.2 | Таблица: 10 колонок | ❌ Отсутствует колонка ЦБС | ... |
   ```
3. **Статусы:** ✅ OK | ❌ Расхождение | ⚠️ Частично | 🔲 Не проверено
4. **Не менять код** — только отчёт. Правки — по отдельной задаче.
5. **Ключевые чек-листы:**
   - [ ] Таблица обращений: 10 колонок (FR-04.2)
   - [ ] Фильтры: 5 кнопок (FR-04.3)
   - [ ] CRM-индикатор: 3 состояния цвета (FR-06.2)
   - [ ] Таймер SLA в шапке карточки (FR-03.1, FR-05.4)
   - [ ] Трёхколоночный layout карточки (FR-04.6—FR-04.8)
   - [ ] Левый сайдбар: заявитель + история + дедлайн + автодозвон (FR-04.7)
   - [ ] Правый сайдбар: комментарии + вложения (FR-04.8)
