# ✅ Реализованные Use Cases для страницы Регистрации

## 📋 Статус реализации

### ✅ UC-1. Автоматический поиск клиента (CRM Integration)

**Реализовано:**
- ✅ Автопоиск при вводе телефона (`onBlur` событие на поле "Номер телефона")
- ✅ Loading индикатор во время поиска (анимированная иконка Loader2)
- ✅ Автозаполнение полей при находке:
  - ФИО Заявителя / Наименование организации
  - Дата рождения / ИНН/КПП
  - Адрес
  - История обращений
- ✅ Автоматическое переключение на "Клиент" при находке
- ✅ Определение VIP-статуса и персонального менеджера
- ✅ Переключение на "Неклиент" если не найдено
- ✅ Toast-уведомления о результатах поиска

**Как работает:**
```tsx
// Поиск запускается при потере фокуса с поля телефона
<input onBlur={handlePhoneBlur} />

// Логика поиска (мок):
const handlePhoneBlur = async () => {
  setIsSearchingCRM(true);
  // Симуляция API запроса в CRM (800ms)
  // Если найден - автозаполнение
  // Если не найден - очистка и переключение на "Неклиент"
  setIsSearchingCRM(false);
}
```

**Мок-логика:**
- Если телефон содержит "999" или "123" → Клиент найден
- "999" → VIP клиент с персональным менеджером
- Другие номера → Неклиент

### ✅ UC-2. Расчет SLA и Приоритета (Автоматический калькулятор)

**Реализовано:**
- ✅ Автоматический расчет плановой даты при выборе типа обращения
- ✅ Автоматическое определение приоритета:
  - **Устное:** +7 дней, Приоритет = Средний
  - **Письменное:** +15 дней, Приоритет = Низкий
  - **Регулятор:** +3 дня, Приоритет = Высокий (заблокирован)
- ✅ Цветовая индикация плановой даты:
  - **Красный (#DD0000):** < 2 дней (очень срочно)
  - **Жёлтый (#FFAA00):** < 5 дней (срочно)
  - **Зелёный (#00AA44):** ≥ 5 дней (норма)
- ✅ Свечение плановой даты соответствующим цветом
- ✅ Иконка календаря окрашена в цвет SLA

**Логика цветов:**
```tsx
const getSLAColor = () => {
  const diffDays = Math.ceil((planned - registered) / (1000 * 60 * 60 * 24));
  
  if (diffDays < 2) return '#DD0000'; // Red
  if (diffDays < 5) return '#FFAA00'; // Yellow
  return '#00AA44'; // Green
}
```

**UI элементы:**
- Приоритет отображается disabled сегментами (нельзя изменить вручную)
- Плановая дата - readonly с динамическим фоном и border
- Текст "🔒 Заблокировано системой" для типа "Регулятор"

### ✅ UC-3. Классификация через Базу Знаний

**Реализовано:**
- ✅ Модальное окно с категориями
- ✅ Выбор категории и подкатегории
- ✅ **Переопределение SLA для критичных категорий:**
  - "Эквайринг → Сбой" = SLA 1 день, Приоритет = Высокий
- ✅ Визуальная индикация критичных категорий (бейдж "SLA 1д")
- ✅ Toast-уведомление при выборе критичной категории

**Критичные категории:**
```tsx
if (category === 'Эквайринг' && subcategory === 'Сбой') {
  daysToAdd = 1; // Override to 1 day!
  newPriority = 'Высокий';
  showToastMessage('⚡ Критичная категория! SLA переопределен на 1 день');
}
```

**Доступные категории:**
1. Кредитование → Начисление процентов
2. Кредитование → Досрочное погашение
3. Карты → Блокировка карты
4. Карты → Ошибка списания
5. **Эквайринг → Сбой** (критичная, SLA 1д)
6. Вклады → Пролонгация вклада
7. Платежи → Ошибка платежа

**Реализовано:**

- ✅ Динамическое изменение текста кнопки:

  - С FCR: "Зарегистрировать и Закрыть (FCR)"
- ✅ Изменение статуса обращения при регистрации:
  - Без FCR: "В работе" / "В пуле"
  - С FCR: "Закрыто (FCR)"
- ✅ Разная логика маршрутизации:
  - FCR → Не попадает в пул, сразу закрыто
  - Есть ПМ → Направлено ПМ

**Success Screen:**
```tsx

  'Обращение решено и закрыто' : 
  'Обращение зарегистрировано'
}

Статус: {fcrChecked ? 'Закрыто (FCR)' : 'В работе'}
```

**Валидация:**
```tsx
const validateForm = () => {
  const newErrors = {};
  
  if (!applicantName && applicantType === 'Физлицо') 
    newErrors.applicantName = true;
  if (!organizationName && applicantType === 'Юр лицо') 
    newErrors.organizationName = true;
  if (!phone) newErrors.phone = true;
  if (!category) newErrors.category = true;
  if (!appealContent) newErrors.appealContent = true;
  
  if (Object.keys(newErrors).length > 0) {
    showToastMessage('❌ Заполните обязательные поля');
    return false;
  }
  return true;
}
```
