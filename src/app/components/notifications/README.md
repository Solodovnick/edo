# Система уведомлений для регистратора

## Описание

Полнофункциональная система уведомлений с колокольчиком для кабинета и карточки регистратора.

## Функционал

### Типы уведомлений

1. **Письменное обращение** (синяя иконка ✉️)
   - Сообщение: "Получено новое письменное обращение для регистрации"
   - Приоритет: обычный
   - Автоматически генерируется при добавлении письменного обращения

2. **Звонок** (красная иконка 📞)
   - Сообщение: "Звонок, выполните регистрацию обращения"
   - Приоритет: высокий (с бейджем "Срочно")
   - Генерируется при входящем звонке

### Компоненты

#### NotificationBell
Основной компонент колокольчика с выпадающим списком уведомлений.

**Props:**
- `variant?: 'cabinet' | 'card'` - вариант отображения (по умолчанию 'cabinet')

**Функции:**
- Отображение количества непрочитанных уведомлений (бейдж с анимацией)
- Выпадающий список с полным функционалом:
  - Просмотр всех уведомлений
  - Отметка как прочитанное (индивидуально)
  - Отметить все как прочитанные
  - Удаление уведомления
  - Очистка всех уведомлений
- Автоматическое обновление каждые 3 секунды
- Toast-уведомления при получении нового сообщения
- Форматирование времени (только что, 5 мин назад, вчера и т.д.)

#### NotificationService
Сервис для управления уведомлениями в localStorage.

**Методы:**
- `getNotifications()` - получить все уведомления
- `addNotification(type: 'письменное' | 'звонок')` - добавить уведомление
- `markAsRead(id: string)` - отметить как прочитанное
- `markAllAsRead()` - отметить все как прочитанные
- `deleteNotification(id: string)` - удалить уведомление
- `clearAll()` - очистить все уведомления
- `getUnreadCount()` - получить количество непрочитанных

### Интеграция

#### В кабинете регистратора (RegistrationPageEnhanced.tsx)
```tsx
import { NotificationBell } from '../notifications/NotificationBell';

// В заголовке
<NotificationBell variant="cabinet" />
```

#### В карточке регистрации (AppealRegistrationCard.tsx)
```tsx
import { NotificationBell } from '../notifications/NotificationBell';

// В заголовке
<NotificationBell variant="card" />
```

### Тестирование

В кабинете регистратора доступны кнопки для симуляции:
1. **"Симулировать входящий звонок"** - создаёт срочное уведомление о звонке
2. **"Добавить письменное обращение"** - создаёт обычное уведомление о письменном обращении

### Особенности

- Хранение в localStorage с ключом `registrar_notifications`
- Максимум 50 последних уведомлений
- Автоматическая инициализация с тестовыми уведомлениями при первом запуске
- Адаптивный дизайн
- Анимации и плавные переход��
- Цветовая кодировка по типам и приоритетам
- Клик вне области закрывает выпадающий список

### Стилизация

- **Кабинет** (`variant="cabinet"`): белая иконка на фиолетовом фоне с прозрачностью
- **Карточка** (`variant="card"`): фиолетовая иконка на светлом фоне
- **Непрочитанные**: выделены фиолетовым фоном
- **Срочные**: красный бейдж "Срочно" с пульсацией
